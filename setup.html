1. Instalacion y configuracion

~15 minutos

Que necesitas

Antes de empezar, confirma que tienes lo siguiente:

  • Stata 17 o posterior, instalado y con licencia
  • Node.js 18 o posterior (descargar)
  • Una API key de Anthropic o una cuenta Claude Max/Pro (pregunta a tu lider de proyecto)
  • Recomendado: una cuenta de GitHub y un repositorio de proyecto clonado
Tip

Ya conoces Git y VS Code? Salta directo a Instalar Claude Code.

NotaQue es GitHub? (expandir si es nuevo para ti)

GitHub es una plataforma donde los equipos almacenan y colaboran en codigo. Es como una carpeta compartida con superpoderes: guarda un historial completo de cada cambio, permite que varias personas trabajen en los mismos archivos sin sobreescribirse, y proporciona un respaldo fuera de tu computador.

En IPA, cada proyecto tiene un repositorio (repo) en GitHub. Un repositorio es una carpeta que contiene todos los do-files, documentacion y configuracion de un proyecto. Cada analista clona (descarga) una copia a su propio computador para trabajar.

Concepto Que es
Repositorio Carpeta de proyecto en GitHub. Contiene do-files, documentacion y configs.
Clonar Descargar una copia del repo a tu computador.
Commit & Push Guardar un snapshot de tus cambios (commit) y subirlos a GitHub (push).

No necesitas ser experto en Git. Estos tres conceptos son suficientes para empezar, y Claude Code puede ayudarte con los comandos.

Estructura de carpetas del proyecto

Los proyectos de IPA deben seguir una estructura estandarizada. La plantilla oficial esta en PovertyAction/ipa-stata-template. Para iniciar un proyecto nuevo, haz clic en “Use this template”.

mi-proyecto/
├── do_files/           # Scripts de Stata
│   └── 00_run.do       # Do-file maestro
├── data/               # Datos crudos, limpios, finales
├── outputs/            # Tablas y graficos
│   ├── tables/
│   └── figures/
├── logs/               # Logs de ejecucion
├── ado/                # Paquetes locales de Stata
├── documentation/      # Guias del proyecto
├── .env                # Ruta local de Stata (no se sube a Git)
└── .here               # Marcador de raiz del proyecto

La plantilla soporta tres niveles: Minimo (solo Git + Stata), Task Runner (agrega just para automatizacion), y Completo (herramientas de Python, pre-commit hooks e integracion con Claude Code).

Tip

Si ya tienes un repositorio de proyecto, no necesitas empezar de cero. La estructura anterior es una referencia.

NotaVS Code: tu espacio de trabajo (expandir si es nuevo para ti)

Visual Studio Code (VS Code) es un editor de codigo gratuito donde se ejecuta Claude Code. Descargalo en code.visualstudio.com.

Panel Funcion Atajo
Barra lateral Estructura de carpetas del proyecto Ctrl+B
Editor Area principal donde se abren archivos como pestanas —
Terminal Linea de comandos integrada. Aqui corre Claude Code Ctrl+`
Extensiones Marketplace de complementos Ctrl+Shift+X

Instala estas extensiones en la terminal de VS Code:

code --install-extension kylebarron.stata-enhanced
code --install-extension anthropic.claude-code
code --install-extension eamodio.gitlens

Instalar Claude Code

npm install -g @anthropic-ai/claude-code
Nota

Error de permisos en Windows? Reabre VS Code como Administrador. En macOS, usa sudo.

Autenticacion

Opcion A: API Key (pago por uso). Estandar para proyectos IPA con facturacion compartida.

# Windows (PowerShell)
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-tu-clave-aqui", "User")
# macOS / Linux (agregar a ~/.zshrc o ~/.bashrc)
export ANTHROPIC_API_KEY="sk-ant-tu-clave-aqui"

Opcion B: Cuenta Claude (Max / Pro). Lanza Claude Code y sigue el prompt de login en el navegador. No se necesita variable de entorno.

Advertencia

Manten tus credenciales privadas. Nunca pegues tu API key en documentos compartidos, Slack, o Git. Las IPA AI Usage Guidelines lo prohiben explicitamente.

Lanzar y verificar

cd /ruta/a/tu/proyecto
claude

Escribe What project is this? para verificar que Claude Code lee tu CLAUDE.md correctamente.

Problemas comunes

Advertencianpm install falla con error de permisos

Windows: Cierra VS Code y reabrelo como Administrador (clic derecho → “Ejecutar como administrador”).

macOS/Linux: Usa sudo npm install -g @anthropic-ai/claude-code. Si el problema persiste, verifica que Node.js se instalo correctamente con node --version.

AdvertenciaClaude Code no reconoce la API key

La variable de entorno no se carga automaticamente en terminales ya abiertas. Despues de configurar ANTHROPIC_API_KEY:

  1. Cierra y reabre la terminal en VS Code (o cierra y reabre VS Code completo)
  2. Verifica con echo $ANTHROPIC_API_KEY (macOS/Linux) o echo %ANTHROPIC_API_KEY% (Windows CMD)
  3. Si sigue vacio, revisa que agregaste la linea al archivo correcto (~/.zshrc, ~/.bashrc, o las variables de entorno del sistema en Windows)
AdvertenciaClaude Code dice que no encuentra el proyecto

Claude Code necesita ejecutarse desde la raiz de tu proyecto — la carpeta que contiene los do-files y, idealmente, un archivo CLAUDE.md.

  1. Verifica que estas en la carpeta correcta: pwd en la terminal
  2. Si no tienes CLAUDE.md, Claude Code aun funciona pero sin contexto del proyecto. Consulta el modulo de conceptos para crear uno.
AdvertenciaLa version de Node.js es muy antigua

Claude Code requiere Node.js 18 o posterior. Verifica tu version con node --version. Si es anterior a 18, descarga la version LTS mas reciente en nodejs.org.

AdvertenciaLas extensiones de VS Code no se instalan con code --install-extension

En algunos sistemas, el comando code no esta en el PATH. Alternativa: abre VS Code, presiona Ctrl+Shift+X, busca cada extension por nombre, y haz clic en “Install”.
