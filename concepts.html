Que es Claude Code?

Claude Code es un asistente de codigo con IA construido por Anthropic. Se ejecuta dentro de la terminal de VS Code. A diferencia de chatbots en el navegador, lee los archivos de tu proyecto, entiende la estructura, y puede escribir o editar codigo directamente en tu maquina.

Tres capacidades lo distinguen:

  1. Contexto del proyecto. Lee tus archivos automaticamente — do-files, configs, documentacion. No necesitas copiar y pegar en una ventana de chat.
  2. Edicion directa. Crea, modifica y elimina archivos en tu proyecto. Pide tu aprobacion antes de hacer cambios.
  3. Instrucciones persistentes. Sigue reglas escritas en un archivo CLAUDE.md en la raiz de tu proyecto — estandares de codigo, convenciones y reglas de seguridad.

Que puede y que no puede hacer

✅ Puede hacer ❌ No puede hacer
Escribir do-files siguiendo estandares IPA/DIME Acceder o leer datos que contengan PII
Diagnosticar errores leyendo logs de Stata Reemplazar tu juicio sobre diseno de investigacion
Generar tablas en LaTeX con esttab o outreg2 Ejecutar Stata por si solo (usa comandos just)
Explicar codigo existente en lenguaje sencillo Acceder a Box, email u otros sistemas externos
Revisar do-files contra estandares de codigo Garantizar metodologia estadistica correcta
Ayudar con comandos de Git Recordar entre sesiones separadas

Como Claude Code conoce tu proyecto

Claude Code no empieza de cero cada vez. Tres mecanismos le dan conocimiento persistente:

CLAUDE.md: el briefing del proyecto

Es como el documento que escribirias para un nuevo miembro del equipo en su primer dia. Le dice a Claude Code que es el proyecto, que reglas seguir, y donde encontrar las cosas.

# CLAUDE.md — Proyecto SpC

## Que es este proyecto
SpC 3.0 = Sanar para Crecer 3.0 (HtG)
Programa de educacion a cuidadores implementado por ICBF en Colombia.

## Reglas del proyecto
- PII: nunca leer o mostrar archivos de datos crudos.
- No usar cap / capture excepto para cap mkdir.
- Globals de rutas definidos en 0_paths.do.

Este archivo se sube a Git. Todos los miembros del equipo comparten las mismas instrucciones.

TipPlantilla CLAUDE.md para proyectos IPA (clic para expandir)

Copia esta plantilla a la raiz de tu proyecto y adapta cada seccion. Para un ejemplo completo, consulta el CLAUDE.md de ipa-stata-template.

# CLAUDE.md

## Descripcion del proyecto
[Nombre del proyecto] — [descripcion de una linea].
Implementado por [organizacion] en [pais/region].

## Reglas criticas
- NUNCA leer, abrir, ni mostrar archivos en `data/raw/` ni datos con PII.
- No usar `capture` excepto para `cap mkdir`.
- Globals de rutas estan definidos en `do_files/00_run.do`.
- Usar convenciones de valores faltantes IPA: .d (don't know), .r (refused),
  .n (not applicable), .s (skipped).

## Estructura del proyecto
- `do_files/`   — Scripts de Stata. `00_run.do` es el maestro.
- `data/`       — Datos crudos, limpios, finales. NO leer raw/.
- `outputs/`    — Tablas (`tables/`) y graficos (`figures/`).
- `logs/`       — Logs de ejecucion de Stata.

## Estandares de codigo
- Seguir IPA Data Cleaning Guide y estandares DIME.
- Variables con prefijos descriptivos (e.g., `bl_` baseline, `el_` endline).
- Headers estandar IPA en cada do-file (proyecto, autor, fecha, proposito).
- Usar `assert` para validacion defensiva.

## Comandos comunes
just stata-run                     # Ejecutar pipeline completo
just stata-script 01_data_cleaning # Ejecutar script individual

Prueba tu CLAUDE.md: abre Claude Code y pregunta “Que proyecto es este?”. Si responde correctamente, tu archivo funciona.

Skills: los manuales de referencia

El directorio .claude/skills/ contiene instrucciones especializadas que Claude Code carga automaticamente segun lo que le pidas. Cada skill es un archivo Markdown con reglas para un dominio especifico.

.claude/
└── skills/
    ├── stata.md          # Estandares IPA/DIME, headers, convenciones
    └── writing.md        # Estilo IPA, voz activa, formato de citas

Por ejemplo, un skill de Stata podria contener:

# Skill: Stata IPA

## Headers de do-file
Cada do-file debe empezar con: proyecto, autor, fecha, proposito.

## Valores faltantes
Usar extended missing values: .d (don't know), .r (refused),
.n (not applicable), .s (skipped). Nunca usar solo "."

## Rutas
Usar globals definidos en 00_run.do. Nunca hardcodear rutas.

Cuando le pides a Claude Code que escriba un do-file, carga automaticamente el skill de Stata. Cuando le pides que redacte un memo, carga el de escritura. No necesitas indicarle cual skill usar.

Modelo mental: CLAUDE.md es el briefing para un nuevo miembro del equipo. Los skills son los manuales de referencia en su escritorio.

Configuracion y permisos

El archivo .claude/settings.json controla que comandos puede ejecutar Claude Code sin pedir permiso. Normalmente lo configura el lider del proyecto.

{
  "permissions": {
    "allow": [
      "just stata-run",
      "just stata-script *",
      "just lint-stata"
    ],
    "deny": [
      "rm -rf *",
      "git push --force"
    ]
  }
}

Los comandos en allow se ejecutan sin pedirte confirmacion. Los de deny estan bloqueados. Todo lo demas te pide aprobacion antes de ejecutarse.
